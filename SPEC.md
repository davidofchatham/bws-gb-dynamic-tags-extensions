# SPEC â€” `src:site` unified site source (v1.9.0, Stage A)

Source: `.claude/plans/src-site-unified-source.md`. Scope: **Stage A only** (base-tag `src:site`). Stage B (#26 try_ slot derive) + Stage C (slot dispatch arm, `class-site.php`) OUT â€” not in this spec.

## Â§G â€” Goal

One source `src:site` for all site-wide data behind existing base tags (text/permalink/image/content/datetime_single/datetime_range). One tag name, `use:` enum picks datum. Unlocks `{{option}}`-style key reads on datetime (ACF date opts) + content (block-markup opts) â€” combos GB Pro lacks.

## Â§C â€” Constraints

- C1 â€” No new source class, no registry registration in Stage A (decision S2). Registering pulls site into `get_effective_sources()` (try_ slot map) before Stage C dispatch arm exists â†’ `resolve_id()`â†’`post_fn(0)` footgun. Class deferred to B/C.
- C2 â€” `site` works as `bws_base_source_option()` dropdown value + early-gate string alone. `bws_resolve_post_by_source` never reached for site (early gate short-circuits before it).
- C3 â€” Option reads OUR responsibility to allowlist-gate. `Meta_Handler::get_option()` enforces only blocklist (`DISALLOWED_KEYS`); does NOT consult `generateblocks_dynamic_tags_allowed_options` (that lives in GB Pro `get_option` callback `class-register.php:257-296`, upstream, skipped when calling handler direct).
- C4 â€” Allowlist seed = **GB Pro parity** (REVISED, was empty A2 â€” see B2). GB Pro's `get_option` callback (verified, GB Pro 2.6.0-beta.2 `class-register.php:268-291`) seeds: `siteurl, blogname, blogdescription, home, time_format, user_count` PLUS `array_keys( GenerateBlocks_Pro_Dynamic_Tags_ACF::get_instance()->get_acf_option_fields() )` (every registered ACF options-page field â€” registration IS the opt-in), then `apply_filters('generateblocks_dynamic_tags_allowed_options', merged)`. We mirror this EXACTLY. ACF-options fields auto-allow (no manual filter). Empty seed (old A2) was a misread of GB Pro that blocked ACF option fields â†’ site option reads returned empty (B2). `{{...src:site|key:blogname}}` now renders out-of-box like `{{option key:blogname}}`.
- C5 â€” Datetime-site = ACF-options read (`get_field($key,'option')`), NOT `get_option`. ACF retains `return_format`; `get_option` loses it. Acceptable â€” datetime tags already ACF-bound everywhere, no NEW lock-in. Non-datetime site values stay ACF-free.
- C6 â€” No `meta`/`single_only` control on site `key`. `get_option` called `$single_only=true` (matches Pro). Array-root reads (no subkey) â†’ empty; useful reads = scalar leaf via dot-path.
- C7 â€” `linkTo` static value list this release (L3a). Conditional JS gates whole controls by `element.key`, not individual `options[]` entries â†’ dropdown shows all 4 values every context. Leakage harmless both ways. Per-value `show_if` gating = L3b, deferred (general capability â†’ #27), do NOT block v1.9.0.
- C8 â€” Content pipeline UNCHANGED. `bws_render_block_content` shipped 1.8.0; Stage A = call site only, no refactor.
- C9 â€” Text domain `'generateblocks'`; all fns prefixed `bws_`; version 1.9.0.
