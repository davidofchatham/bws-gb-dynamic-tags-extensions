# SPEC ‚Äî `src:site` unified site source (v1.9.0, Stage A)

Source: `.claude/plans/src-site-unified-source.md`. Scope: **Stage A only** (base-tag `src:site`). Stage B (#26 try_ slot derive) + Stage C (slot dispatch arm, `class-site.php`) OUT ‚Äî not in this spec.

## ¬ßG ‚Äî Goal

One source `src:site` for all site-wide data behind existing base tags (text/permalink/image/content/datetime_single/datetime_range). One tag name, `use:` enum picks datum. Unlocks `{{option}}`-style key reads on datetime (ACF date opts) + content (block-markup opts) ‚Äî combos GB Pro lacks.

## ¬ßC ‚Äî Constraints

- C1 ‚Äî No new source class, no registry registration in Stage A (decision S2). Registering pulls site into `get_effective_sources()` (try_ slot map) before Stage C dispatch arm exists ‚Üí `resolve_id()`‚Üí`post_fn(0)` footgun. Class deferred to B/C.
- C2 ‚Äî `site` works as `bws_base_source_option()` dropdown value + early-gate string alone. `bws_resolve_post_by_source` never reached for site (early gate short-circuits before it).
- C3 ‚Äî Option reads OUR responsibility to allowlist-gate. `Meta_Handler::get_option()` enforces only blocklist (`DISALLOWED_KEYS`); does NOT consult `generateblocks_dynamic_tags_allowed_options` (that lives in GB Pro `get_option` callback `class-register.php:257-296`, upstream, skipped when calling handler direct).
- C4 ‚Äî Allowlist seed = **GB Pro parity** (REVISED, was empty A2 ‚Äî see B2). GB Pro's `get_option` callback (verified, GB Pro 2.6.0-beta.2 `class-register.php:268-291`) seeds: `siteurl, blogname, blogdescription, home, time_format, user_count` PLUS `array_keys( GenerateBlocks_Pro_Dynamic_Tags_ACF::get_instance()->get_acf_option_fields() )` (every registered ACF options-page field ‚Äî registration IS the opt-in), then `apply_filters('generateblocks_dynamic_tags_allowed_options', merged)`. We mirror this EXACTLY. ACF-options fields auto-allow (no manual filter). Empty seed (old A2) was a misread of GB Pro that blocked ACF option fields ‚Üí site option reads returned empty (B2). `{{...src:site|key:blogname}}` now renders out-of-box like `{{option key:blogname}}`.
- C5 ‚Äî Datetime-site = ACF-options read (`get_field($key,'option')`), NOT `get_option`. ACF retains `return_format`; `get_option` loses it. Acceptable ‚Äî datetime tags already ACF-bound everywhere, no NEW lock-in. Non-datetime site values stay ACF-free.
- C6 ‚Äî No `meta`/`single_only` control on site `key`. `get_option` called `$single_only=true` (matches Pro). Array-root reads (no subkey) ‚Üí empty; useful reads = scalar leaf via dot-path.
- C7 ‚Äî `linkTo` static value list this release (L3a). Conditional JS gates whole controls by `element.key`, not individual `options[]` entries ‚Üí dropdown shows all 3 values (none/permalink/key) every context. Leakage harmless both ways. Per-value `show_if` gating = L3b, deferred (general capability ‚Üí #27), do NOT block v1.9.0. NO `linkTo:site` value ‚Äî site permalink-analog folds into `linkTo:permalink` (V-link).
- C8 ‚Äî Content pipeline UNCHANGED. `bws_render_block_content` shipped 1.8.0; Stage A = call site only, no refactor.
- C9 ‚Äî Text domain `'generateblocks'`; all fns prefixed `bws_`; version 1.9.0.

## ¬ßI ‚Äî Interfaces

- I.src ‚Äî `bws_base_source_option()` (base-tags.php ~557): add `{value:'site', label:'Site'}`.
- I.resolve ‚Äî new `bws_site_resolve_value($tag, $options, $instance)`. Used by text/title/permalink/image/content ONLY (not datetime). **NO `use:option` value** (dropped, B3). Resolve order: (1) named site `use:` value ‚Üí its datum; (2) else ‚Üí site option key-read. This is NOT "use unset = key" globally ‚Äî it is `src:site` specifically routing any non-named-site `use` (including each tag's own default, e.g. content's `content`) into the option read, because under `src:site` there is no post entity to apply a post-default to.
  - NO site `use` enum on any tag (V9 ‚Äî dropped `logo`/`home_url`/`site_url` as redundant with bare analogs; `site_url()` intentionally unreachable this release). Site model = **bare ‚Üí best analog, `key` ‚Üí option**.
  - Per-tag bare analog (no key): title‚Üí`get_bloginfo('name')`; text‚Üí`get_bloginfo('name')`? NO ‚Äî text keyed-by-nature, bare=empty (V9); content‚Üí`get_bloginfo('description')` (B4); permalink‚ÜíALWAYS `home_url()` (source's own URL; `key` IGNORED, no option read ‚Äî narrowed, see V9 + T16); image‚Üílogo: `(int)get_theme_mod('custom_logo')` ‚Üí `bws_get_attachment_data($id, as ?? 'url', size ?? 'full')` (empty/no logo ‚Üí '').
  - title tag has NO `use` enum ‚Üí `$tag==='title'` (or `use:title`) ‚Üí `get_bloginfo('name')`.
  - content site PARALLEL (B4): bare `{{content src:site}}` (no `key`) ‚Üí `get_bloginfo('description')` (tagline = blogdescription), paralleling post‚Üícontent / term‚Üídescription / site‚Üídescription. content with `key` set ‚Üí option key-read (rich-option render path). So content's named site datum IS the description; `tagline` is therefore REMOVED from the text `use` enum (single-path tagline killed ‚Äî reach it via `{{content src:site}}`).
  - option key-read (the non-named branch) + `key` set ‚Üí allowlist-gate (C4 GB-parity), then `Meta_Handler::get_option($key,true,'')`. Content: pass raw through `bws_render_block_content($raw,'option:'.$key)` instead (NO extra `wp_kses_post`). Empty `key` ‚Üí for content = tagline (above); for text/image = `''`. permalink does NOT participate in the option key-read at all (returns `home_url()` before this branch ‚Äî `key` ignored, no `key` control).
- I.gate ‚Äî text/content/title/permalink/image callbacks (base-tags.php 725/797/841/898/932): early gate `if ('site'===($options['src']??'')) return bws_site_resolve_value(...)` before entity-resolve. title callback site path must still apply `bws_wrap_with_link(...,1,'site')` (link-eligible).
- I.dt ‚Äî `bws_base_datetime_single_callback` (datetime-tags.php:988) / `_range_callback`: SEPARATE path. On `src===site` call `bws_datetime_single_core('option', bws_base_map_datetime_options($options), $instance)` (+range equiv), then `bws_wrap_with_link(...,1,'site')` tail. NO `use` enum ‚Äî `key` is direct field-key control.
- I.read ‚Äî **DT-1**: `bws_read_field` (field-helpers.php:~230, before `return null` tail) add: `if('option'===$post_id && function_exists('get_field')){ allowlist-gate $key; return get_field($key,'option'); }`. Sentinel currently dead-ends at null ‚Üí zero behavior change for int/loop/term callers.
  - **DT-1b (caller seam, ‚ÜíB1/V7)**: `bws_parse_combined_date_time` (datetime-helpers.php:272/280) passes `$numeric_id` to `bws_read_field`, NOT raw `$post_id`. `$numeric_id = is_numeric($post_id)?(int):false`, so `'option'`‚Üí`false` and the DT-1 sentinel never arrives. MUST thread the `'option'` sentinel to the value read: `$value_id = ('option'===$post_id) ? 'option' : $numeric_id;` then pass `$value_id` at both date+time reads. Non-`'option'` callers unchanged (still `$numeric_id`).
- I.link ‚Äî **L3**: `bws_resolve_link_url` (link-helpers.php) `'site'` is an `entity_type`, NOT a `linkTo` value. `linkTo:permalink`+`entity_type==='site'`‚Üí`home_url()` (site permalink-analog; ignores `$id`); `linkTo:key`+site‚Üíallowlist-gate `$link_key` then `get_option($link_key)`. Site callbacks pass `$link_type='site'`, sentinel `$link_id=1` (defeats `if(!$id)` guard). NO `linkTo:site` value in dropdown ‚Äî permalink already IS the site canonical URL (no duplicate path per datum). Link options on text/title/datetime_* only (`supports_link_wrap`); content/permalink/image excluded.
- I.opts ‚Äî per-tag `use:` enum. **NO `option` value anywhere** (B3). Site named-data are `use:<not key>` entries; the default key-mode (empty `use`) + `key` set = option read.
  - NO site `use` values on any tag (V9). text: `use` enum = existing `title`,`key` (no site-specific value; bare text+site = empty, keyed) | permalink: NO `use` enum AND NO `key` control under site ‚Äî bare = `home_url()` ALWAYS (source's own URL, not an option read; narrowed ‚Äî see V9/T16. URL-valued options reachable via `{{text src:site|key:...}}`) | image: `use` enum = existing `featured`,`key` (NO `logo`; bare image+site = logo) | content: NO site `use` value ‚Äî bare = description (B4), `key` = option | title: NO use enum (site‚Üíname) | datetime_single/range: NO use enum.
  - `key` shown for site whenever key-mode: extend each tag's existing `key` `show_if` to also show under `src:site` in key-mode. text key `show_if not_in:title,tagline` already covers it (empty/key/option-namespace all key-mode). permalink has NO `key` control AT ALL (option-key read dropped ‚Äî permalink site = `home_url()` only). image key `show_if not_in:featured,logo`. content key `show_if in:key` PLUS site (use empty under site = content default is post-content; site needs key ‚Üí see note). title + datetime: datetime existing `key` always-visible (direct field/option key); title no key.
  - NOTE content (B5-corrected, Model B): content's default `use` = `content` resolves the analog in EVERY source ‚Äî post content / term description / **site tagline** (`get_bloginfo('description')`). Site-content is NOT forced to key-mode; `use:key` reads the option, `use:content` (default) ‚Üí tagline. `key` `show_if` for content = `show_if {use:key}` (NOT the old `show_if_any{use:key,src:site}`, which was the B5 over-reach that showed the key field for the tagline default under site).
  - Suppress for site (`show_if src:not:site`) ‚Äî on ALL site-capable tags incl. title:
    - `srcTermIn` ‚Äî conceptually N/A for site (no entity to hop terms from).
    - `ref` ‚Äî suppressed in Stage A because there is NO site‚Üíref wiring yet (early gate short-circuits before `bws_resolve_post_by_source`; resolver switches on `use` only). NOT "ref never applies to site" ‚Äî ACF relational fields stored as site options are plausible (untested). Re-expose `ref` (drop the gating) when a site‚Üíref resolution path ships ‚Üí **#28**. (Mechanically `ref` keeps its existing `show_if src:ref` which already hides it for site ‚Äî no new gate needed.)
- I.adr ‚Äî [ADR 0001](docs/adr/0001-site-option-read-allowlist.md): **REWRITTEN** (B2) ‚Äî GB-parity allowlist (6 WP defaults + ACF option-field keys + filter), "OUR resolver gates not handler" (still true), 3 gated read paths (site option key-mode, site `linkTo:key`, datetime `get_field(‚Ä¶,'option')`). Empty-seed premise was a misread of GB Pro ‚Äî corrected.

## ¬ßV ‚Äî Invariants

- V1 ‚Äî `bws_site_allowlist_ok($key)` MUST gate every option read with the **GB-parity seed** (REVISED, B2): `$seed = ['siteurl','blogname','blogdescription','home','time_format','user_count']; if (class_exists('GenerateBlocks_Pro_Dynamic_Tags_ACF')) $seed = array_merge($seed, array_keys(GenerateBlocks_Pro_Dynamic_Tags_ACF::get_instance()->get_acf_option_fields())); $allowed = apply_filters('generateblocks_dynamic_tags_allowed_options', $seed); $parent = explode('.',$key)[0]; return in_array($parent,$allowed,true);`. NOT empty seed (that misread of GB Pro blocked ACF option fields, B2). !C3,C4,ADR 0001 ‚Äî `@invariant` PHPDoc on `bws_site_allowlist_ok`.
- V2 ‚Äî All THREE option-read paths gate through `bws_site_allowlist_ok` (same GB-parity filter): site option key-mode read, site `linkTo:key`, datetime `get_field(‚Ä¶,'option')`. Consistent disclosure boundary. ACF keys flat ‚Üí gate whole `$key` (dot-path split only matters for wp_options arrays; `$parent` = first segment handles both). **Single-reader corollary (B4):** the wp_options value paths (key-mode value read + `linkTo:key`) MUST share ONE reader `bws_site_read_option($key)` (field-helpers.php) ‚Äî gate + `Meta_Handler::get_option` (dot-path + ACF filter). (Datetime path reads ACF FIELDS via `get_field(‚Ä¶,'option')`, a different datum ‚Äî separate reader, same gate.) `@invariant` PHPDoc on `bws_site_read_option`.
- V3 ‚Äî Datetime-site output formatting MUST flow through `bws_build_single_format` / `bws_build_range_format`. Site-format fallback (`get_option('date_format')`/`time_format`) = their tier-3; MUST NOT be re-implemented independently. - V4 ‚Äî Stage A introduces NO source class + NO registry registration. `site` is dropdown-value + early-gate-string only. !C1,C2.
- V5 ‚Äî Site early gate MUST pre-empt any `SourceRegistry::get_source('site')` lookup (returns null ‚Üí choke). Preview-label + GB-registration paths must not call it for site. Editor preview = likely edge.
- V6 ‚Äî `content` site option-read recursion bounded by existing 1.8.0 guard via `'option:'.$key` cache key (key collision = guard hit, by design). Distinct keys bounded by `bws_content_max_recursion_depth` (default 3). No pipeline change. !C8.
- V7 ‚Äî Site-datetime value read MUST receive the `'option'` object-id sentinel. `bws_parse_combined_date_time` passes a value-read id to `bws_read_field` that is `'option'` when `$post_id==='option'`, else `$numeric_id` (existing behavior). The format-lookup path (`$acf_object_id` via `bws_resolve_acf_object_id`) already resolves `'option'` correctly; the VALUE-read path is a separate arg and was NOT threading it (B1). !I.read,DT-1b ‚Äî drove B1.
- V9 ‚Äî **Source-analog resolution.** SCOPE: base-tag callbacks and try_ **slot 1** only ‚Äî i.e. wherever an empty `use`/`key` wire value means "the template default" (the strip-default-first-value position). Does NOT govern try_ slot ‚â•2: there an empty wire = "same as previous slot" (carry-forward to a concrete inherited token, see V8), which never re-derives an analog. Within scope: when a tag resolves to its DEFAULT `use` token it SHOULD yield the best intrinsic analog datum for the active source, WHERE ONE EXISTS. Analogs (current): title‚Üí{post title, term name, site name}; content‚Üí{post content, term description, site description}; permalink‚Üí{post URL, term URL, site `home_url()`}; image‚Üí{post featured, ‚Äî(term has none ‚Äî gap, #29 may fill via configurable default key), site logo}; text‚Üíkeyed by nature in ALL contexts (default = key, key required). Where a source has no intrinsic analog (term image today, site text-body), default = empty + key required (honest gap).
  - **DISPATCH MECHANISM (B5-corrected, Model B).** `use` is the analog-vs-option selector ‚Äî UNIFORM across ALL sources incl. site. The lever is the `use` VALUE, NOT key-presence. `bws_site_resolve_value` MUST dispatch on `use` per tag; it MUST NOT branch the analog on `'' === $key` (that was B5 ‚Äî a misapplied future-control principle that made `use` dead under site + showed an enum of ignored post/term values). Per-tag site dispatch: content `use:content`(default)‚Üítagline (`get_bloginfo('description')`), `use:key`‚Üíoption, `use:excerpt`‚Üí'' (no site excerpt; entry stays visible until per-value filter T17/#27); image `use:featured`(default)‚Üílogo (`get_theme_mod('custom_logo')`, respects `as`/`size`), `use:key`‚Üíoption attachment-id; text `use:title`‚Üísite name, `use:key`+key‚Üíoption, bare/empty‚Üí'' (keyed by nature, no analog default ‚Äî unchanged); title/permalink no `use` enum. The analog datum a source exposes IS reached by its DEFAULT `use` token (content's `content`, image's `featured`) ‚Äî so "bare tag ‚Üí analog" still holds, but BECAUSE the default `use` value dispatches to the analog, not because key is empty.
  - **permalink** stays narrowed: names the source's OWN URL (a relation to the entity, not a field), so under `src:site` it is `home_url()` with NO `use` enum and NO `key` control (option-key read dropped ‚Äî T16). URL-valued options live behind `{{text src:site|key:...}}`.
  - **COROLLARY (redundant SITE-SPECIFIC values):** a NEW site-only `use:` value that would duplicate a datum already reachable via an existing tag's default analog is redundant and SHOULD NOT exist ‚Äî drove dropping `use:logo`/`use:home_url`/`use:site_url` (logo = image default `featured`; home_url = permalink; site_url unreachable this release). This is NOT "no site `use` enum" (that was the B5 over-reach) ‚Äî the EXISTING per-tag `use` values dispatch under site; only NEW redundant site-specific values are barred. NOTE: a concrete analog default (e.g. site image‚Üílogo via `use:featured`) is also what try_ slot ‚â•2 carry-forward inherits, so concrete > empty for slot reuse. !I.resolve,I.opts,B4,B5,#29. ‚Üí migrate to CONTEXT.md post-ship.
