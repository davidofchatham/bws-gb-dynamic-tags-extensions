## ¬ßV ‚Äî invariants

- V1. **Factory precedence: `loop_ctx` ‚Üí `get_queried_object()` ‚Üí current-post; `$post`/`get_the_ID()` NEVER ambient fallback.** Probe-proven: `$post` carries main query's first row on EVERY results-bearing non-singular context (term archive 48418, search 47955, empty-search 51604). Binds for Phase-1 kinds (post/term/site/meta_row/registry); contexts without a kind fall through to current behavior. Owner: I.engine.
- V2. **Resolved source = flat array `{kind + kind-specific keys}`; unknown/malformed kind ‚Üí `array()` silent-empty, no crash.** `{'kind'=>'post','id'=>N}` / `{'kind'=>'term','id'=>N}` / `{'kind'=>'meta_row','row'=>arr}` / `{'kind'=>'site'}`. Typedef PHPDoc on `bws_run_traversal()` ONLY. Owner: I.engine.
- V3. **Seam freeze: `bws_resolve_field_values()` signature + raw `string[]` return unchanged; every existing caller renders identically** (except V6 plural). Owner: I.seam.
- V4. **Wrapper compat: `bws_resolve_post_by_source()` returns first post id | false, byte-compatible across all 39 call-sites.** Wrapper callers stay collapse-to-first ‚Äî plural reaches seam consumers only. Owner: I.dispatch.
- V5. **Portal renders identically with ZERO portal changes:** `base_source_key:'view'` resolves via registry delegation; `ref` step replaces `PortalRelatedPost` traversal; `traversal_source_key` accepted-but-ignored (never rejected, never required). Owner: I.registry.
- V6. **`src:ref` plural: seam consumers read N ref targets; `limit` default 1 ‚Üí output unchanged unless author set `limit>1`.** Fan-out via ref step, no `bws_extract_post_id` collapse inside pipeline. Owner: I.seam, I.engine.
- V7. **Term ambient: bare base tags on term archive (outside loop) resolve the TERM ‚Äî analogs per CONTEXT.md I1 (title‚Üíname, content‚Üídescription, permalink‚Üíterm URL), key reads ‚Üí `bws_read_term_field()`, ALL base tags incl. text/datetime/email/phone. Explicit options ALWAYS beat ambient.** Non-analog gaps stay honest-empty (image ‚Üí empty, I1 gap #29). Owner: I.engine, I.dispatch.
- V8. **try_ output byte-identical EXCEPT the term-ambient parity gain (T8) ‚Äî a slot resolves IDENTICALLY to the same base tag used standalone (I6/C9).** A try_ slot on a term archive reads the term analog exactly as the T6 base callback does; every non-term-archive slot output is unchanged. The `try_core_fn`/`try_term_fn` fork STAYS (its collapse is deferred future-work) ‚Äî parity is achieved by the slot's base-source resolution, NOT by merging the dispatch fns. Owner: I.registry.
- V9. **Engine pure + deterministic: no side effects, empty-step passthrough, short-circuit on first empty step, fan-out preserves document order.** Owner: I.engine.
- V10. **Factory + resolver sit DOWNSTREAM of try_ slot carry-forward ‚Äî never re-derive analogs or apply slot-position semantics.** `generate_base_try_tags` assembles `$slot_opts` first (`$last_src`/`$last_ref`/`$last_key`/`$last_use` merged for slot ‚â•2 inherit; registry.php:697-712) THEN calls the resolver. Slot-1 strip-default-first vs slot-‚â•2 carry-forward (CONTEXT.md I1) is UPSTREAM option-assembly; the factory receives fully-materialized `$slot_opts` + dispatches purely on them (no slot notion) ‚Äî re-deriving would double-apply. `srcTermIn` read per-slot, NEVER carried. Binds T8 fork collapse: preserve assemble-then-resolve ORDER; factory stays downstream. Owner: I.registry (assembly), I.engine (purity).
- V11. **`src:ref` on a term archive bases the ref-hop on the AMBIENT TERM (term ‚Üí post[] step), NOT on `$post`.** The term is the ambient resolved source (V7); a ref step hops its relationship field off the term ‚Äî V7 applied to the ref case, NOT deferred. This FIXES a live leak: today `RelatedPost::resolve_id` reads the rel field from `GenerateBlocks_Dynamic_Tags::get_id($options,'post')` = `get_the_ID()` = the stale first-loop post on an archive (probe 48418), so `src:ref` on an archive currently reads a relationship field off an arbitrary leaked post. Factory does NOT gate term-ambient by src ‚Üí ref included. Confined to term archives: on singular pages `queried_kind` is null ‚Üí post base, byte-unchanged (no regression, V3/V4 safe). The deferred parity gap (`traversal-pipeline.md` ¬ßProblem) is PINNING a SPECIFIC non-ambient primary (a chosen term/post ID THEN hop) ‚Äî unrelated to ambient-term-as-base. Owner: I.engine. (Drove B1.)
- V18. **A post-kind resolved source with id 0 reads EMPTY at the seam ‚Äî never falls to `bws_read_field` inference (extends V12).** `bws_read_resolved_source` post case guards `return $id > 0 ? bws_read_field( $key, $instance, $id ) : '';`. A `{kind:'post', id:0}` means the factory found NO current post; reading a field off post 0 is meaningless. Passing 0 to `bws_read_field` would FAIL V12's explicit-id bypass ‚Äî its guard `$has_explicit_post_id = (is_int && >0) || (is_numeric && (int)>0)` treats 0 as non-explicit, so it re-runs its OWN loop/term inference and can re-derive a context the factory rejected (the "two layers fight" case). The narrow live edge: factory yields post/0 (is_tax/is_category/is_tag false, get_the_ID()==0) yet `get_queried_object()` is a `WP_Term` ‚Üí `bws_read_field`'s un-gated `instanceof` reads term meta (B7). NOT the V17 case (that is is_tax-true-but-no-term; this is the inverse). Guarding id 0 at the seam closes it with NO change to `bws_read_field` (its ~30 non-seam callers keep their own inference). Owner: `bws_read_resolved_source`. (Drove B7.)
- V17. **A claimed-taxonomy-context with NO resolvable term yields EMPTY, never the current/leaked post (extends V1).** When `is_tax()`/`is_category()`/`is_tag()` report a taxonomy archive but `get_queried_object()` is not a `WP_Term` (null / non-term ‚Äî term deleted after the query-object cache, malformed query, `pre_get_posts` manipulation), `bws_capture_ambient_signals` sets `'term_context_unresolved' => true`. `bws_resolve_base_source`, AFTER the loop/explicit/ambient-term branches and BEFORE the current-post fallback, returns `array()` (¬ßV2 silent-empty) ‚Äî a bare base tag on a claimed term archive must NOT leak the main query's first post (the ¬ßV1 leak, degenerate case). Explicit src (site/registry/ref) and loop rows precede the flag ‚Üí they still resolve; only the BARE ambient read on a broken term context short-circuits to empty. Owner: `bws_capture_ambient_signals` + `bws_resolve_base_source` PHPDoc. (Drove B6.)
- V16. **`src:ref` post hop is ACF-COMPATIBLE, not ACF-mandatory: an ACF relationship/post_object field (type-validated, plural) OR any meta value holding a post id / list of ids (raw).** The post/ref reader (`bws_pipeline_default_reader`) tries `bws_get_related_posts_data` FIRST ‚Äî ACF-only, `get_field_object` type-guarded to relationship|post_object, returning the plural relationship array. On empty (ACF absent, field not a relationship, or a non-ACF handler storing ids in plain meta ‚Äî Pods/Carbon/core) it FALLS BACK to `bws_extract_post_id( get_post_meta( $id, $field, true ) )` ‚Äî the raw non-ACF path OLD Mode-2a used. A non-ACF site (or plain-meta-post-id field) resolves `src:ref` again; an ACF site keeps the type-safe plural path. Honors the ACF-or-compatible contract (CLAUDE.md; reads `function_exists`-guarded). The raw fallback is shape-protected by the `bws_pipeline_ref_to_posts` guard (a string-keyed assoc is ONE value, not a fabricated per-scalar list ‚Äî review #2). Scope: POST-kind ref only; term/user readers already read raw (their non-ACF story is separate). Owner: `bws_pipeline_default_reader` PHPDoc. (Drove B5.)
- V15. **Term-image resolves the configured FALLBACK even with no key ‚Äî standalone `{{image}}` == try_image slot (I6/C9).** The #29 term-image gap means no intrinsic analog DATUM, NOT "no output": a Media Library fallback is last-resort and applies gap-or-not. `bws_base_term_analog_read('image')` calls `bws_term_custom_image_core` UNCONDITIONALLY (no key gate) ‚Äî that core owns the no-key path (empty key ‚Üí `bws_handle_term_image_fallback` ‚Üí fallback, or '' when none set). No key + no fallback ‚Üí '' on the frontend (honest gap preserved); no key + fallback ‚Üí the fallback ‚Äî byte-identical to the try_image term-ambient arm, which calls the same core. Gating the standalone on `'' !== $key` (dropping the fallback) is the B4 defect. Owner: `bws_base_term_analog_read` PHPDoc (permanent home; this ¬ßV is transient ‚Äî migrates/deletes post-ship). Cross-ref I6 (try_ transparency). (Drove B4.)
- V14. **Offered ‚ü∫ resolvable at the LIST-MODE level: a base tag that EXPOSES `limit`/`sep` for a source (via the control's `show_if`/`show_if_any`) MUST honor list mode for that source.** text + title offer `limit`/`sep` for `src:ref` (base-tags.php:105-117/184-196, comment "list mode ‚Ä¶ related posts (src:ref)"), so BOTH must read the FULL fanned-out ref post set and list-join (slice-to-limit, `implode($sep)`, single-result link-wrap only) ‚Äî NOT collapse-to-first. The ref step already fans out (`bws_run_traversal` returns ALL targets, ¬ßV6); the callback must consume the plural list, not `bws_first_post_id_from_sources()` it. **This extends V6 plural to the BASE text/title tags** ‚Äî the plan's "base tags stay collapse-to-first, plural = seam consumers only" fence is SUPERSEDED for a tag that advertises the control (contradiction-refactorable: the offered affordance is the intent, the collapse is the bug). `permalink`/`image` DON'T offer `limit` for src:ref (attribute / single-value destination, I7); `content` is body/not-joinable (I7) ‚Äî those correctly stay collapse-to-first. ¬ßV4 (wrapper byte-compat) is UNAFFECTED: the wrapper stays first-only for its non-callback consumers; only the text/title CALLBACK src:ref branch reads the list directly (not via the wrapper). Owner: I.dispatch. (Drove B3.)
- V13. **Wrapper step-set is REF-ONLY ‚Äî `bws_resolve_post_by_source()` NEVER assembles a `srcTermIn` step.** The wrapper is post-semantic (¬ßV4): hop `src:ref` (post‚Üípost[]) then collapse to first post id. `srcTermIn` (post‚Üíterm) is OWNED DOWNSTREAM by callers ‚Äî datetime/text/title srcTermIn branches call `bws_get_srcterm_terms($post_id, $tax)` on the returned POST id, applying limit/sep/link themselves. A wrapper srcTermIn step would hop post‚Üíterm ‚Üí `bws_first_post_id_from_sources` sees kind 'term' ‚Üí false ‚Üí caller's `bws_get_srcterm_terms(false)` ‚Üí [] ‚Üí empty (B2). The SEAM (`bws_resolve_field_values`) DIFFERS: srcTermIn IS a terminal step there (term-list read mode, ¬ßV6/¬ßV12) ‚Äî the seam reads term fields by kind; the wrapper cannot (contract = post id). Split assembly: seam = ref + srcTermIn (`bws_field_values_assemble_steps`); wrapper = ref only. Do NOT route the wrapper through `bws_field_values_assemble_steps`. Owner: I.dispatch. (Drove B2.)
- V12. **Seam dispatches L2 by resolved-source KIND ‚Äî factory owns source-selection, per-kind reader owns the read; NO double resolution.** `bws_resolve_field_values`: factory ‚Üí `{kind,id}` ‚Üí dispatch `site`‚Üí`bws_site_read_option`, `term`‚Üí`bws_read_term_field`, `post`‚Üí`bws_read_field` with an EXPLICIT `$post_id`. The explicit id triggers the v1.7.1 explicit-wins rule (field-helpers.php:242), bypassing `bws_read_field`'s OWN loop/term-archive inference (lines 271-296) so the factory's resolved source is authoritative ‚Äî the two resolution layers never fight. `bws_read_field`'s internal fallback stays LIVE for its ~30 non-seam callers (reached via the wrapper, T5) but is inert on the seam path. Owner: I.seam. Regression risk: a term read routed through `bws_read_field` WITHOUT an explicit id would re-derive context and could diverge from the factory ‚Äî always pass the id / route term‚Üí`bws_read_term_field`.
